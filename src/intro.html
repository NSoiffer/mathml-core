Rationale for a new version of MathML

This section is non-normative.

The [auto] specification has several shortcomings that make it hard to implement consistently accross web rendering engines:

  1. It is a huge and standalone specification.
  2. It does not contain any detailed rendering rules.
  3. It is not driven by browser-implementation.
  4. It lacks automated testing.

This MathML Core specification intends to address these issues by being as accurate as possible on the visual rendering of mathematical formulas using additional rules from the TeXBookâ€™s Appendix G [auto] and from the Open Font Format [auto]. It also relies on modern browser implementations and web technologies [auto], extending them or clarifying interactions when needed. Parts of MathML 3 that do not fit well in this framework or are rarely used in practice have been ommited and moved into a larger specification [auto].

By increasing the level of implementation details and by focusing more on browser-driven design, this specification is expected to greatly improve MathML interoperability. A test suite is available in the auto repository.

Terminology
